# Building and maintaining Koralyte

## Requirements

* Hugo Extended v0.146.0+
* Node.js 22+
* Go 1.22+

## Rebuild process

```
npm install     # updates node_modules and package-lock.json
npm run assets  # wipes assets/lib and copies vendor packages to assets/lib
npm run build   # rebuild minified CSS
```

## Adding a shortcode

Shortcodes can be added by adding an HTML file to `layouts/_shortcodes`. Hugo will automatically resolve these when
referenced in content like

```hugo
{{< newshortcode key="value" >}}
```

If the new shortcode needs a vendored JS/CSS library, add a conditional load in `layouts/_partial/vendor.html` so that
the new libraries are only included in pages that use the shortcode. Afterward, add a corresponding `devVendorCopy`
entry in `package.json`.

## Updating vendored libraries

Third-party JS/CSS libraries (KaTeX, Fuse.js, etc.) are stored in `assets/lib`. These are copied from `node_modules/` by
the `vendor-copy` postinstall step.

To update them, you change the version in `package.json` and rebuild the theme.

Every entry in `devVendorCopy` should point to a library that is still in `devDependencies` (and used somewhere in the
theme's templates).

## Updating Tailwind

Tailwind is pulled via the various `tailwind` and `@tailwind` entries in `devDependencies`. These should be kept on
consistent versions and bumped together.

It is worth performing a quick visual sanity check whenever updating Tailwind as it affects the styling across the
entire theme.

## Auto-generated files

There are several files in the theme that are autogenerated and should not be edited.

* The compiled CSS in `assets/css/compiled/main.css`
* The vendored third-party libraries in `assets/lib`

## Hugo module release

The Hugo module is declared in `go.mod`. It's generally good practice to tag releases so that users can pin these to a
specific release.

```
hugo mod get github.com/ragibson/koralyte@v1.0.0
```

## Security

In general, we'd like to keep dependencies to a minimum (and possibly move to a safer alternative than npm). In the
meantime, it's best to audit the dependencies periodically.

```
npm audit
npm audit fix          # apply "safe" fixes
npm audit fix --force  # fixes that allow major version bumps
```

## Directory layout

The purpose of the larger theme directories are below.

```
koralyte/
├── assets/
│   ├── css/
│   │   ├── compiled/      # (automatic) generated main.css
│   │   ├── components/    # chroma.css for code syntax highlighting
│   │   ├── schemes/       # color schemes
│   │   └── main.css       # Tailwind and theme styling rules
│   ├── icons/             # SVG icons (social and theme UI)
│   ├── js/                # theme javascript utilities
│   └── lib/               # (automatic) vendored libraries
├── layouts/               # flat layouts structure as in modern Hugo (v0.146.0+)
│   ├── _markup/           # render hooks
│   ├── _partials/         # theme partials
│   ├── _shortcodes/       # theme shortcodes
│   ├── baseof.html        # base template
│   ├── home.html          # home page kind
│   ├── page.html          # single page kind
│   ├── section.html       # section listing
│   ├── taxonomy.html      # taxonomy listing
│   ├── term.html          # term page
├── go.mod                 # Hugo module path
├── hugo.toml              # theme defaults
├── theme.toml             # Hugo theme metadata
├── tailwind.config.js     # Tailwind v4 config (uses @config shim)
└── package.json           # npm scripts and setup
```
