# Building and maintaining Koralyte

## Requirements

- Hugo Extended v0.158.0+
- Node.js 22+
- pnpm 11+
- Go 1.22+

## Rebuild process

```
pnpm install       # updates node_modules and pnpm-lock.yaml (postinstall also re-copies vendor files)
pnpm assets        # wipes assets/lib, re-copies vendor files, and regenerates the Chroma syntax CSS with Hugo
pnpm build         # rebuild minified CSS
pnpm format        # format all files
pnpm format:check  # check formatting (for CI)
```

## Code formatting

Koralyte uses two formatters:

- Prettier for JS, CSS, JSON, YAML, and Markdown.
- gotmplfmt for Go/Hugo templates in `layouts/`.

gotmplfmt is a standalone CLI, so it has to be installed separately (it is not managed via package.json/go.mod):

```bash
go install github.com/gohugoio/gotmplfmt@latest
```

For now, gotmplfmt does not sort Tailwind classes inside `layouts/` (unlike `prettier-plugin-tailwindcss` in JS/CSS
files), so we should try keeping class ordering in the templates reasonably consistent by hand.

## Adding a shortcode

Shortcodes can be added by adding an HTML file to `layouts/_shortcodes`. Hugo will automatically resolve these when
referenced in content like

```hugo
{{< newshortcode key="value" >}}
```

If the new shortcode needs a vendored JS/CSS library, add a conditional load in `layouts/_partials/vendor.html` so that
the new libraries are only included in pages that use the shortcode. Afterward, add a corresponding `vendor` entry in
`package.json`.

## Updating vendored libraries

Third-party JS/CSS libraries (KaTeX, Fuse.js, etc.) are stored in `assets/lib` and refreshed by `pnpm assets`. Most of
these are copied from `node_modules/` by `scripts/vendor-copy.js`. The Chroma syntax-highlighting CSS is instead
generated by `scripts/generate-chroma.js`.

To update a library, change its version in `package.json`'s `devDependencies` and re-run `pnpm install` and
`pnpm assets`.

Every entry in `vendor` should point to a library that is still in `devDependencies` (and used somewhere in the theme's
templates).

## Updating Tailwind

Tailwind v4 is pulled via the `@tailwindcss/*` packages in `devDependencies` (`@tailwindcss/cli` bundles the matching
`tailwindcss` release), plus `prettier-plugin-tailwindcss` for class sorting. These should be kept on consistent
versions and bumped together.

It is worth performing a quick visual sanity check whenever updating Tailwind as it affects the styling across the
entire theme.

## Auto-generated files

There are several files in the theme that are autogenerated and should not be edited.

- The compiled CSS in `assets/css/compiled/main.css`
- The vendored third-party libraries and generated Chroma CSS in `assets/lib`
- `pnpm-lock.yaml` (regenerated by pnpm when dependencies change)

## Hugo module release

The Hugo module is declared in `go.mod`. It's generally good practice to tag releases so that users can pin these to a
specific release.

The Go version in the module should be kept up to date with the rest of the documentation.

```
hugo mod get github.com/ragibson/koralyte
```

## Security

In general, we'd like to keep dependencies to a minimum. It's also best to audit the dependencies periodically.

```
pnpm audit
pnpm audit --fix
```

## GitHub Actions

There are two GitHub Actions workflows containing five tests.

- `build.yaml` ensures that the theme builds properly on a simple, mock site. This is repeated for the minimum Hugo
  version in `hugo.toml` and the latest one.
- `generated-files.yaml`'s `css-up-to-date` job recompiles the theme's CSS and ensures it matches what has been
  committed to the theme.
- `generated-files.yaml`'s `vendor-up-to-date` job similarly ensures that the theme's committed vendor files are up to
  date.
- `generated-files.yaml`'s `format-check` job ensures that the theme's auto-formatter has been run.

The minimum Hugo version is declared in `hugo.toml`. When updating it, also update the version in `build.yaml`'s matrix
and the requirements section of this document and `README.md`.

## Directory layout

The purpose of the larger theme directories are below.

```
koralyte/
├── archetypes/            # default front matter templates for content
├── assets/
│   ├── css/
│   │   ├── compiled/      # (automatic) generated main.css
│   │   ├── components/    # code block layout styles (chroma-layout.css)
│   │   ├── schemes/       # color schemes
│   │   └── main.css       # Tailwind and theme styling rules
│   ├── icons/             # SVG icons (social and theme UI)
│   ├── js/                # theme javascript utilities
│   └── lib/               # (automatic) vendored libraries and generated Chroma CSS
├── i18n/                  # translation files (English-only in the theme)
├── layouts/               # flat layouts structure as in modern Hugo (v0.146.0+)
│   ├── _markup/           # render hooks
│   ├── _partials/         # theme partials
│   ├── _shortcodes/       # theme shortcodes
│   ├── 404.html           # 404 error page
│   ├── baseof.html        # base template
│   ├── home.html          # home page kind
│   ├── index.json         # JSON output for client-side search
│   ├── list.html          # default list layout
│   ├── page.html          # single page layout
│   ├── robots.txt         # robots.txt template
│   ├── rss.xml            # RSS feed template
│   ├── section.html       # section listing
│   ├── sitemap.xml        # sitemap template
│   ├── taxonomy.html      # taxonomy listing
│   ├── term.html          # term page
├── scripts/               # build scripts
├── static/                # static files
├── go.mod                 # Hugo module path
├── hugo.toml              # theme defaults
├── theme.toml             # Hugo theme metadata
├── tailwind.config.js     # Tailwind v4 config (uses @config shim)
├── package.json           # pnpm scripts and setup
├── pnpm-workspace.yaml    # pnpm workspace settings
├── .npmrc                 # enforces the Node/pnpm engine versions in package.json
├── .editorconfig          # editor formatting defaults
├── .prettierrc            # Prettier formatting config
├── .prettierignore        # Prettier formatting ignore rules
```
